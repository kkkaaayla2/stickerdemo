# 横评墙（Comparison Wall） · PRD

> **文档版本**：v1.0

---

## 0. 命名 & 一句话卖点

- **功能名（产品向）**：横评墙
- **运营 / 玩梗名**：贴贴投票 · 夯拉墙
- **一句话卖点**：贴出你的态度

---

## 1. 问题聚焦：为什么要做

### 1.1 现状

小红书现有投票仅支持发布「**1 个问题 × N 个选项**」且用户只能「**N 选 1**」。见【原其他用户投票前界面.jpg】

### 1.2 三方痛点

- **作者**：想做横评，没有结构化容器；笔记发出后评论区脏乱、难统计、难二次复盘
- **读者**：想表达态度，只能N选1，但可能想要多选或者表达“又爱又恨”
- **平台**：决策类 UGC 价值大量沉淀在散落的评论里，无法被结构化抽取、聚合、二次分发

### 1.3 机会点

把线下"贴纸投票墙"的物理参与感（参考【线下投票墙.jpg】）搬到线上：撕、拖、贴、堆叠——用**游戏化交互**降低参与门槛，让"问大家"从单选题变成一场围观感强烈的群体决策。同时绑定「**夯/拉**」这种网络梗，形成产品记忆点。

- **决策协助型**：自己面临同类选择，看群体结果做参考
- **吃瓜围观型**：被"夯 vs 拉"的对立感和贴纸交互吸引而沉浸

| 产品                | 形态       | 多对象支持 | 交互方式    |
| ----------------- | -------- | ----- | ------- |
| 小红书 现有投票          | 单议题 N 选项 | ❌     | 点击单选    |

**结论**：「**多对象横评 + 物理化拖拽贴纸** 」的组合形态，在主流内容社区中尚未出现，差异化空间充足。

## 4. 产品逻辑（抽象）

### 4.1 数据模型

```
ComparisonWall {
  question: string                        // 题目
  objects: Object[2..6]                   // 评价对象，2 ≤ N ≤ 6
  options: [RedOption, BlueOption]        // 固定 2 个对立选项
                                          // 默认 "夯"/"拉"，用户可自己填
                                          // 文案可改、颜色固定红蓝、不可增减
  votes: Vote[]                           // 每条投票 = {userId, objectId, optionId}
}
```

### 4.2 核心规则


| 规则项      | 设定                                                   |
| -------- | ---------------------------------------------------- |
| 对象数量     | 2-6 个                                                |
| 选项数量     | 固定 2 个对立选项；文案可改（如 行/不行、买/不买）；颜色固定红蓝；不可增减             |
| 单用户贴纸额度  | 红 × N + 蓝 × N（N = 对象数）                               |
| 单对象单用户上限 | 红 × N + 蓝 × N（N = 对象数）（**允许"又爱又恨"同时投，允许全部贴纸投给一个对象**） |
| 结果触发条件   | 投出第 **1** 张贴纸即解锁全部结果（蒙层消失）                           |
| 剩余贴纸     | 保留在贴纸栏，可继续贴或退出，剩余数量退出后下次仍可继续                         |
| 回访状态     | 直接显示已投结果；用户**自己刚投的那张贴纸高亮**（脉冲 / 发光）                  |
| 未投查看     | 「查看结果」按钮直接消蒙层；**查看后仍可正常投票**                          |
| 票数视觉     | 模拟堆叠（demo 阶段不做大数量级降级）                                |
| 作者权限     | **作者本人可投自己笔记的票**，票数计入"X 人已参与"                        |


### 4.3 布局规则（2 列固定）

无论对象数量多少，统一采用 **2 列网格**布局：


| 对象数 | 布局            | 排列说明                   |
| --- | ------------- | ---------------------- |
| 2   | 2 × 1         | 1 行 2 个                |
| 3   | 2 列，第 2 行 1 个 | 第 2 行单格居中              |
| 4   | 2 × 2         | 2 行各 2 个               |
| 5   | 2 列，第 3 行 1 个 | 前 2 行各 2 个 + 第 3 行单格居中 |
| 6   | 2 × 3         | 3 行各 2 个               |


固定 2 列保证每个区域宽度一致、视觉重心稳定，对应手机竖屏阅读体验。

## 5. 主要功能

### 5.1 发布端

#### 5.1.1 模式切换

现有「投票」按钮入口下（如【原投票按钮入口.jpg】所示），进入投票创建页后顶部以两个 Tab 并列展示：

- **「经典模式」** ← 原有的单议题 N 选项投票
- **「横评模式」** ← 本次新增的横评墙投票

#### 5.1.2 题目输入

自由输入框

#### 5.1.3 对象添加

默认显示 2 个对象输入框，下方"+ 添加对象"按钮最多扩展到 6 个；每个对象名称限制 ≤ 8 字。

#### 5.1.4 选项设置

默认"夯/拉"；文案可改（如 行/不行、买/不买、买/不买），但**不可增减数量、不可改颜色**（红/蓝固定）。

#### 5.1.5 横评图自动生成

根据对象数量按 4.3 布局规则生成图片：每区域为浅色块 + 居中对象名。发布后图片栏会比作者上传时多 1 张系统生成的横评图。

同时会显示“投票·2个选项”，和原有界面一致，如【原选项设置完成入口.jpg】

#### 5.1.6 完成发布

原发布逻辑保留；点击「完成」插入笔记，「发布笔记」走原流程。

### 5.2 浏览端

#### 5.2.1 初始状态

- 横评图覆盖**闪烁蒙层**，文案提示"撕张贴纸投票，看大家的选择"
- 下方贴纸盒：红色"夯" × N + 蓝色"拉" × N（N = 对象数）
- 贴纸有撕痕、阴影等仿真细节，呈现出"可撕下"的物理质感

#### 5.2.2 撕拖交互

1. 长按贴纸 → 撕下（轻微旋转 + 阴影 + 触感反馈）
2. 拖动贴纸到目标区域
3. 松手 → 贴纸落入区域内 → 完成投票

#### 5.2.3 首张触发

第 1 张贴纸贴出后：

- **所有区域**蒙层同时消失
- 各区域以**模拟堆叠**形式展示既有票（红蓝贴纸视觉堆叠）
- 每个区域下方小字显示「夯:X / 拉:Y」

#### 5.2.4 持续投票

- 贴纸盒对应数量 -1
- 退出笔记后剩余贴纸数量保留，下次进入可继续贴

#### 5.2.5 本人投票标识

用户自己投出的贴纸保持**高亮 / 脉冲微动效**（如发光边框、轻微缩放呼吸），让 ta 在群体票里能一眼找到自己。

#### 5.2.6 底部信息条

- 「X 人已参与」（含作者本人）
- 评论入口（保留原有样式）

#### 5.2.7 未投查看

保留原有「查看结果」按钮，点击后直接消除蒙层；查看后贴纸盒**仍可继续操作**正常投票。

### 5.3 Demo 阶段数据规则

- 每个对象可预置初始基数（例：麻六记 夯 10 / 拉 20）
- 用户拖入贴纸后实时 +1（夯 10 / 拉 21）
- 不考虑千 / 万级数量的视觉降级，留作后续迭代

## 6. 使用流程

### 6.1 发布流程

```
【原初始发布界面.png】所示界面
    ↓
点击「投票」按钮
    ↓
切换至「横评模式」Tab
    ↓
填题目 → 添加 2-6 个对象 →（可选）改选项文案
    ↓
点击「完成」
    ↓
系统自动生成横评图并插入图片栏已有图片的下一位
    ↓
点击「发布笔记」
```

### 6.2 首次投票流程

```
刷到笔记
    ↓
看到带蒙层的横评墙 + 下方贴纸盒
    ↓
长按撕下贴纸 → 拖到目标对象 → 松手投票
    ↓
所有区域蒙层消失，结果全显（模拟堆叠 + 数字）
    ↓
（可选）继续贴 / 评论 / 退出
```

### 6.3 二次访问流程

```
再次进入笔记
    ↓
直接看到自己已投的状态（自己的贴纸高亮）
    ↓
剩余贴纸仍可继续贴
    ↓
可评论、可退出
```

## 7. 验证方式：如何评估方案生效

### 7.1 北极星指标

**「横评墙笔记发布量」 × 「横评墙投票参与率」**

同时衡量供给（作者愿不愿用）和消费（读者愿不愿玩）。

### 7.2 关键指标拆解


| 维度  | 指标         | 计算方式                 | 目标值（初版灰度）          |
| --- | ---------- | -------------------- | ------------------ |
| 供给  | 横评模式渗透率    | 横评笔记数 / 含投票笔记总数      | ≥ 15%              |
| 供给  | 作者复用率      | 30 天内发过 ≥ 2 次的作者占比   | ≥ 25%              |
| 消费  | 投票转化率      | 投出 ≥ 1 张贴纸的浏览者 / 浏览者 | ≥ 8%（对比单议题基线 +50%） |
| 消费  | 平均贴出贴纸数    | 单用户单笔记平均贴纸数          | ≥ 2.5 张            |
| 消费  | "查看结果"未投占比 | 直接看结果未投票的浏览者占比       | 监测，理想 < 投票转化率      |
| 互动  | 评论率提升      | 横评 vs 单议题投票评论率 Δ     | +20%               |
| 互动  | 收藏 / 转发提升  | 横评 vs 单议题投票互动率 Δ     | +10%               |


### 7.3 验证手段

1. **A/B Test**：发布器入口对 50% 作者灰度新功能；对照组保持旧版投票。
2. **行为漏斗**：
  - 发布漏斗：进入投票创建 → 选模式 → 填对象 → 完成 → 发布
  - 投票漏斗：浏览 → 看到贴纸盒 → 撕下 → 贴出 → 持续贴

### 7.4 风险监测指标（不做兜底功能，仅监测）

- **刷票**：单设备短时间内异常贴纸密度
- **手势冲突**：撕下未贴出比例（识别拖拽是否与上下滑动冲突）
